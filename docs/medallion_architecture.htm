Medallion Architecture

Bronze · Silver · Gold Data Pipeline

FEMA National Flood Insurance Programme  ·  PySpark on Databricks

Module:    7006SCN · Coventry University

Platform:  Databricks Community Edition

Engine:    Apache PySpark 4.0

Records:   2,765,442 FEMA claims

Pattern:   Lakehouse Medallion Architecture

Figure 1.  Medallion Architecture implemented for the FEMA flood insurance analytics project. Raw CSV data is ingested and progressively refined through three named layers —

 Bronze (raw validated), Silver (cleaned and enriched), and Gold (ML-ready feature store) — before being consumed by 14 machine learning algorithms across three tasks. Each layer is persisted as a Parquet file.

SILVER

Tier 2  —  Cleaned & Enriched Data  |  Feature engineering · Outlier treatment · Encoding · Scaling

 

 

🔧 Feature Engineering

+ amountPaidRatio = building / contents

+ Flood zone risk score (A=5, B=3, X=1)

+ claimYear, claimMonth from dateOfLoss

+ propertyAge from year built to loss year

+ high_payout binary label (top 25% threshold)

📉 Outlier Treatment

+ IQR bounds computed per numeric col

+ Values < Q1-1.5*IQR capped at lower fence

+ Values > Q3+1.5*IQR capped at upper fence

+ Z-score > 4 flagged as is_extreme column

+ Window functions used (no collect to driver)

🏷️ Categorical Encoding

+ StringIndexer: floodZone → floodZoneIdx

+ StringIndexer: occupancyType → occTypeIdx

+ StringIndexer: state → stateIdx

+ OneHotEncoder applied after indexing

+ Encoder pipeline fitted on training data only

⚖️ Normalisation & Assembly

+ StandardScaler (mean=0, std=1) on numerics

+ VectorAssembler combines all feature cols

+ Output: dense feature vector of 13 dimensions

+ features column created (MLlib-ready)

+ Pipeline.fit() on train; transform on test

2.  Delta Lake Storage Properties

Each medallion layer is persisted as a Delta table on DBFS. Delta Lake provides ACID transactions, schema enforcement, and time-travel capabilities that are critical for a reliable multi-layer pipeline.

🔒 ACID Transactions

Atomic writes to each medallion layer. Partial failures do not corrupt existing data.

🔄 Schema Evolution

Bronze → Silver → Gold schema changes tracked. Old queries continue to work.

⏪ Time Travel

VERSION AS OF and TIMESTAMP AS OF allow rollback to any prior layer state.
